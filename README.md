# Проект 1: **Клавиатурный тренажёр**  

### Описание проекта
К сожалению, не все студенты и преподаватели Центрального университета могут печатать быстро и без ошибок.
Чтобы это исправить, тебе нужно создать клавиатурный тренажёр, который поможет людям развивать навык слепого набора текста, повышая продуктивность при написании кода, документов и общении с чатом GPT.

### Задачи проекта

**Реализовать интерактивную виртуальную клавиатуру:**

- отображение нажатий клавиш в реальном времени при вводе с физической клавиатуры;
- поддержка двух раскладок: русская и английская;
- визуальное выделение нажатых клавиш на экране.

**Организовать систему текстовых наборов для печати:**

- возможность добавлять и удалять текстовые задания через модальное окно;
- возможность выбора текста для тренировки через боковое меню;
- поддержка как русских, так и английских текстов.


**Реализовать систему отслеживания производительности:**

- счётчик скорости печати в реальном времени;
- счётчик ошибок в реальном времени;
- отведённый под конкретное задание таймер на выполнение задания.

**Показать статистику после завершения тренировки:**

- модальное окно с итоговой статистикой: количество ошибок, средняя скорость, точность, оценка;
- кнопки «Повторить» и «Выбрать другое задание».

### Спецификация поведения

#### Фича 1: виртуальная клавиатура с подсветкой нажатий

Виртуальная клавиатура визуально отражает нажатия с физической клавиатуры, помогая пользователям ассоциировать клавиши с их расположением. Фича важна для новичков, обучающихся слепому набору. Поддерживает две раскладки - русскую и английскую.

##### Требования к UI/UX

- Отображается под текстом для печати.
- Каждая клавиша - отдельный `div` с текстом и `data-key`-атрибутом.
- При нажатии клавиши на физической клавиатуре соответствующая клавиша на виртуальной подсвечивается (добавляется класс `pressed`).
- Подсветка снимается через 200–300 мс или при отпускании клавиши (событие `keyup`).
- При ошибке (нажата не та клавиша) - кратковременная подсветка красным (`error`).
- Клавиатура отображает актуальную раскладку в зависимости от выбранного текста (или автоматически по языку текста).
- Адаптивно масштабируется: на мобильных - вся клавиатура помещается в контейнер с горизонтальной прокруткой при необходимости.
- Используются `aria-label` для доступности (например, `aria-label="клавиша A"`).

##### Функциональные требования

- Каждая клавиша в разметке имеет `data-key` с кодом клавиши (например, `KeyA`, `Space`, `Enter`).
- Обработка событий: `keydown` и `keyup` на `document`.
- Сопоставление `event.code` с элементами виртуальной клавиатуры.
- Поддержка специальных клавиш: `Shift`, `Backspace`, `Space`, `Enter`, `CapsLock` (визуально отображаются, но не влияют на текст).
- Переключение раскладки происходит автоматически при выборе текста на русском или английском.
- При переключении раскладки DOM клавиатуры обновляется (смена текста на клавишах).

**Тесты**

У следующих HTML-элементов необходимо указать атрибут `automation-id` для прохождения тестов:
- **клавиша виртуальной клавиатуры**:
    - `automation-id="keyboard-key"`;
- **обёртка виртуальной клавиатуры**:
    - `automation-id="keyboard-container"`.

##### Технические ограничения

- Никаких внешних шрифтов для клавиш - использовать системные.
- Вся клавиатура генерируется в TS.
- Клавиатура рендерится динамически на основе объекта-шаблона (например, массив строк с клавишами).

#### Фича 2: текст для печати и отслеживание ввода

Пользователь видит текст, который необходимо набрать. Система отслеживает ввод, подсвечивает текущий символ, выделяет уже введённые и ошибки. Это основа тренировочного процесса.

##### Требования к UI/UX

- Текст отображается в центре экрана, крупным шрифтом.
- Каждый символ - отдельный `span` с классом:
    - `default` - ещё не введён;
    - `current` - ожидается ввод;
    - `correct` - введён верно;
    - `error` - введён с ошибкой.
- Текущий символ подсвечен (например, жёлтый фон).
- Ошибки подчёркиваются красным.
- При пропуске символа он помечается как ошибка.
- Под текстом - индикатор прогресса в процентах.

##### Функциональные требования

- Текст разбивается на символы при загрузке задания.
- При вводе каждый символ сравнивается с ожидаемым.
- Курсор визуально не нужен - фокус на отслеживании потока.
- При достижении конца текста - тренировка завершается досрочно (если таймер ещё идёт).
- Поддержка пробелов, знаков препинания, заглавных букв (с учётом Shift).

**Тесты**

У следующих HTML-элементов необходимо указать атрибут `automation-id` для прохождения тестов:
- **контейнер текста для печати**:
    - `automation-id="text-container"`;
- **отдельный символ в тексте**:
    - `automation-id="text-symbol"`;
- **текущий символ, который нужно ввести**:
    - `automation-id="current-symbol"`;
- **символ, введённый корректно**:
    - `automation-id="correct-symbol"`;
- **символ, введённый с ошибкой**:
    - `automation-id="error-symbol"`;
- **индикатор прогресса (в процентах)**:
    - `automation-id="progress-bar"`.

##### Технические ограничения

- Нельзя использовать `contenteditable` или `input` для ввода - фокус на `document`.
- Ввод блокируется до выбора задания.
- После завершения ввод игнорируется до открытия статистики.

#### Фича 3: система таймера и счётчиков

Таймер запускается при начале тренировки, а счётчики показывают скорость и количество ошибок в реальном времени. Это мотивирует пользователя улучшать результаты.

##### Требования к UI/UX

- Таймер отображается в виде цифрового счётчика (например, `1:34`).
- Стартует при первом нажатии клавиши.
- Скорость печати обновляется каждую секунду: `Скорость: 45 зн/мин` (округлённое до целого).
- Счётчик количества ошибок изменяется в реальном времени: `Ошибки: 3` (счётчик увеличивается при ошибке).
- Все счётчики обновляются динамически.

##### Функциональные требования
- Таймер - `setInterval` с обновлением каждую секунду.
- Скорость рассчитывается как `количество правильно введённых символов / прошедшее время в минутах`.
- Ошибки инкрементируются при несовпадении символа.
- При досрочном завершении (текст закончен) - таймер останавливается, статистика фиксируется.

**Тесты**

У следующих HTML-элементов необходимо указать атрибут `automation-id` для прохождения тестов:
- **счётчик скорости печати**:
    - `automation-id="speed-counter"`;
- **счётчик количества ошибок**:
    - `automation-id="error-counter"`;
- **таймер тренировки**:
    - `automation-id="timer-counter"`.

##### Технические ограничения

- Никаких внешних библиотек для таймера.
- Все вычисления - в TypeScript.

#### Фича 4: управление наборами текстов (боковое меню и модальное окно)

Пользователь может выбирать, добавлять и удалять текстовые задания. Это позволяет кастомизировать тренировки под свои цели.

##### Требования к UI/UX
- Боковое меню слева.
- Список паков: название, язык, кнопка удаления (иконка корзины).
- Кнопка «Добавить задание» - открывает модальное окно.
- Модальное окно: поля «Название», «Текст», «Язык» (выпадающий список: Русский/English), кнопки «Сохранить», «Отмена».
- При наведении на пак - подсветка, клик - запуск тренировки.
- Удаление - через модальное подтверждение.

##### Функциональные требования
- Все паки хранятся в памяти (массив объектов в TS).
- При добавлении - валидация полей.
- При удалении - проверка, что остаётся хотя бы один пак.
- После добавления - новый пак появляется в списке.
- При клике на пак - загружается текст, обновляется интерфейс.
- Учитывать время прохождения при начале тренировки (использовать именно из данных текстового набора).

##### Валидация

- При попытке добавить пустой текст: `"Пожалуйста, заполните все поля"`.
- При попытке добавить задание без названия: `"Пожалуйста, заполните все поля"`.
- При удалении последнего задания: `"Нельзя удалить последний пак"`.

**Тесты**

У следующих HTML-элементов необходимо указать атрибут `automation-id` для прохождения тестов:
- **боковое меню**:
    - `automation-id="sidebar"`;
- **элемент списка текстовых наборов**:
    - `automation-id="text-pack-item"`;
- **кнопка «Добавить задание»**:
    - `automation-id="add-text-button"`;
- **модальное окно добавления задания**:
    - `automation-id="add-text-modal"`;

    - `automation-id="input-title"`;
- **поле ввода «Текст» в модальном окне**:
    - `automation-id="input-text"`;
- **поле выбора «Язык» в модальном окне**:
    - `automation-id="input-language"`;

    - `automation-id="cancel-button"`;

- **модальное окно подтверждения удаления**:
    - `automation-id="delete-confirmation-modal"`;
- **кнопка «Удалить» в модальном окне подтверждения**:
    - `automation-id="modal-confirm-button"`;
- **кнопка «Закрыть» в модальном окне подтверждения**:
    - `automation-id="modal-close-button"`.

##### Технические ограничения

- Модальное окно генерируется динамически через JS.
- Модальные окна управляются через CSS-классы (например, `modal-open`).

#### Фича 5: модальное окно статистики

После завершения тренировки пользователь видит итоговую статистику: скорость, ошибки, точность, оценка. Это даёт обратную связь и мотивацию.

##### Требования к UI/UX
- Центрированное модальное окно с затемнённым фоном.
- Заголовок: `Тренировка завершена!`.
- Поля:
    - `Задание: [название] `;
    - `Скорость: 52 зн/мин`;
    - `Ошибки: 2`;
    - `Точность: 96%`;
    - `Оценка: Хорошо` (по шкале: Плохо, Удовл., Хорошо, Отлично).
- Кнопки: `Повторить`, `Выбрать другое задание`.
- Закрытие по клику вне окна или на крестик.

##### Функциональные требования
- Статистика рассчитывается при завершении:
    - точность = `(правильные / общее количество символов) * 100`;
    - оценка определяется по диапазонам:
        - `< 70%` - `Плохо`;
        - `70–84%` - `Удовлетворительно`;
        - `85–94%` - `Хорошо`;
        - `≥ 95%` - `Отлично`.
- При нажатии `Повторить` - перезапускается то же задание, сброс таймера и счётчиков.
- При нажатии `Выбрать другое задание` - закрывается модальное окно, возвращается в меню выбора.

**Тесты**

У следующих HTML-элементов необходимо указать атрибут `automation-id` для прохождения тестов:
- **модальное окно статистики**:
    - `automation-id="stats-modal"`;
- **заголовок модального окна**:
    - `automation-id="stats-modal-title"`;
- **поле «Задание» в статистике**:
    - `automation-id="stats-field-name"`;
- **поле «Скорость» в статистике**:
    - `automation-id="stats-field-speed"`;
- **поле «Ошибки» в статистике**:
    - `automation-id="stats-field-errors"`;
- **поле «Точность» в статистике**:
    - `automation-id="stats-field-accuracy"`;
- **поле «Оценка» в статистике**:
    - `automation-id="stats-field-rating"`;
- **кнопка «Повторить»**:
    - `automation-id="repeat-button"`;
- **кнопка «Выбрать другое задание»**:
    - `automation-id="choose-another-button"`.

##### Технические ограничения
- Модальное окно генерируется динамически через JS.
- Все данные передаются напрямую из объекта тренировки.
- Никаких анимаций через сторонние библиотеки - только CSS-анимации/переходы/RAF.
- При открытии модального окна - блокировка прокрутки фона (`document.body.style.overflow = 'hidden'`).

#### Фича 6: поддержка двух языков и раскладок

Приложение поддерживает тренировку как на русском, так и на английском языке. При выборе текста автоматически подгружается соответствующая виртуальная клавиатура.

##### Требования к UI/UX
- В интерфейсе отображается индикатор языка: `RU` или `EN` рядом с названием задания.
- При выборе текста на русском виртуальная клавиатура переключается на кириллицу.
- При выборе английского - на латиницу.
- Шрифты поддерживают оба алфавита (без подключения внешних).

##### Функциональные требования
- Виртуальная клавиатура перерисовывается при смене задания.
- Раскладка определяется по полю `language` в объекте пака (`'ru'` | `'en'`).
- При вводе с физической клавиатуры - сравнение символов идёт по фактическому вводу (`event.key`), а не по раскладке.
- Поддержка заглавных букв при зажатом `Shift` или `CapsLock`.

**Тесты**

У следующих HTML-элементов необходимо указать атрибут `automation-id` для прохождения тестов:
- **индикатор языка (рядом с названием задания)**:
    - `automation-id="language-indicator"`.

##### Технические ограничения
- Никакого определения раскладки через ОС - только визуальное отображение.
- Символы сравниваются по значению (`event.key === expectedChar`).
- При ошибках - не учитывается раскладка, только факт несовпадения.

### Оформление и разработка проекта

#### Правила коммитов
- Формат: `<тип>: <описание>`.
- Примеры:
    - `feat: добавлена виртуальная клавиатура`;
    - `fix: исправлен подсчёт ошибок`;
    - `style: форматирование кода`;
    - `docs: обновлена документация проекта`.
- Разрешённые типы: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`.

#### Ветки
- Основная ветка: `main`.
- Фичи разрабатываются в ветках: `feature/<название>` (например, `feature/virtual-keyboard`).
- Исправления: `fix/<описание>`.
- Все изменения - через Merge Request (MR).  
- Финальная версия проекта сдается через MR в `main` ветку.

#### Проверки
- Код должен компилироваться через `tsc` без ошибок.
- Наличие типов для всех переменных и функций.
- Запрещён `any`, `// @ts-ignore`, `!` (non-null assertion) без обоснования.
- HTML и CSS - валидные, без предупреждений.

#### Запуск проекта
- Проект запускается с помощью `vite`.
